Q. CPU와 직접 통신하는 기억 장치는?

📖 개념의 정의

기억 장치는 프로그램과 데이터를 저장했다가 꺼내 쓰는 공간이에요.

  • 주기억: CPU와 직접 데이터를 주고받는 메모리
  • 보조기억: 당장 안 쓰는 것을 오래 보관, 필요할 때 주기억으로 옮김
  • 접근 시간: 읽고 쓰는 데 걸리는 시간
  • 기억 용량: 얼마나 담을 수 있는지
  • 휘발성: 전원 OFF → 내용 삭제 (RAM) / 비휘발성: 남음 (ROM, SSD)

🍱 쉬운 비유 — 학교 급식

CPU = 너의 식판, 주기억 = 급식 카운터(오늘 메뉴만), 보조기억 = 냉동 창고(다음 주 메뉴 보관).

기억 장치 계층 피라미드
레지스터 · 가장 빠름
캐시
주기억 (RAM)
보조기억 (SSD/HDD)
테이프 등 — 가장 느림·큼
컴퓨터는 ‘빠른 메모리 + 큰 창고’를 단계적으로 씁니다.

Q. 전원을 꺼도 내용이 남는 기억은?

Q. 계층에서 보통 가장 아래(느리고 큼)에 가까운 것?

📖 개념의 정의

주기억은 실행 중인 프로그램·데이터가 올라가는 작업 공간이에요.

  • 시스템 영역: 운영체제(OS)
  • 사용자 영역: 게임, 브라우저 등 우리 앱
  • 단일 사용자 할당: 한 사람이 책상 전체 사용 (단순, 낭비 가능)
  • 고정·가변 분할: 여러 프로그램을 칸막이·맞춤 크기로 배치
  • 기억 집약: 빈 칸을 모아 큰 공간 만들기 (정리 중엔 잠깐 멈춤)

📚 쉬운 비유 — 동시에 여러 과제하는 책상

책상이 작으면 과제가 조각(단편화) 나서 들어가기 어렵고, 기억 집약은 책상 위를 싹 정리해 빈 곳을 한곳으로 모으는 거예요.

주기억 = 지금 돌아가는 프로그램의 ‘책상’입니다.

🧪 미니 실습 — 주기억 분할·할당

총 256칸 주기억에 프로그램을 넣어 보세요. 알고리즘별로 빈 칸(단편화)이 달라져요.

🟣 OS(64) 🔵 프로그램 ⬜ 빈 칸

💡 「적재하기」를 1번 이상 누르면 실습 미션이 완료돼요.

Q. 운영체제가 저장되는 영역은?

📖 개념의 정의

종류쉬운 말특징
RAM지우개 있는 작업판읽기·쓰기 자유, 휘발성, 주기억으로 사용
DRAM충전지 컵저렴·대용량, 재충전 필요
SRAM빠른 개인 서랍비쌈·소용량, 캐시에 사용
ROM인쇄된 교칙비휘발성, 읽기 위주
플래시수정 가능한 공책USB·SSD의 기반

☕ 쉬운 비유

DRAM = 공용 충전 스테이션(자주 충전), SRAM = 선생님 서랍(즉시 꺼냄), ROM = 학교 교칙 책(항상 있음).

📊 DRAM vs SRAM 비교 (시험 빈출!)
항목DRAM (동적)SRAM (정적)
저장 방식커패시터(충전)플립플롭
재충전필요 (주기적)불필요
속도느림빠름 (약 5배)
집적도높음 (대용량)낮음 (소용량)
가격저렴비쌈
주 용도주기억 장치캐시 기억 장치

💡 ROM의 종류: Mask ROM(변경 불가) → PROM(1회 기록) → EPROM(UV 삭제·재기록) → EEPROM(전기 삭제) → Flash Memory(블록 단위 입력, HDD 대용)

RAM은 ‘지우개 있는 작업판’, ROM은 ‘남는 설명서’입니다.

Q. 캐시에 주로 쓰이는 빠른 RAM은?

📖 개념의 정의

캐시는 주기억 데이터 일부를 CPU 가까이 복사해 두는 고속 완충 기억이에요.

  • 적중(Hit) = “찾았다!” → 빠름
  • 실패(Miss) = 없음 → 주기억에서 블록 단위로 가져옴 → 느림
  • 참조의 지역성: 자주 쓰는 일부만 반복 접근 → 캐시가 효과적
  • 적중률이 높을수록 컴퓨터가 빨리 느껴짐
CPU가 데이터를 찾는 순서
CPU → 캐시 확인 → 적중 ✓ 또는 실패 ✗ → 주기억

📝 쉬운 비유 — 시험 전날 예상 문제 노트

전과목 교과서(주기억)를 다 읽기엔 늦어요. 자주 나올 단원만 노트(캐시)에 적어 두고, 시험장에서 노트에 있으면 바로 풀어요!

적중률 90% → 평균 접근 시간: 85 ns

막대가 길수록 체감 속도 ↑

🧪 미니 실습 — 직접 사상 계산

주기억 블록 번호 j, 캐시 슬롯 수 m → 슬롯 번호 i = j mod m

슬롯 번호 i = 13 mod 8 = 5
🔬 심화: 사상(매핑) 3가지
  • 직접 사상: 정해진 서랍 칸에만 넣기 (간단)
  • 연관 사상: 아무 빈 칸 (빠르지만 비쌈)
  • 집합 연관: 칸 묶음 안에서 선택 (절충)
🔄 교체 알고리즘 (Cache Replacement)

캐시가 꽉 찼을 때 새 데이터를 넣으려면 기존 데이터를 빼야 합니다. 누구를 빼는지 결정하는 방법:

알고리즘기준비유
LRU가장 오래 안 쓴 것을 제거오래 안 입은 옷 버리기
FIFO가장 먼저 들어온 것 제거줄 맨 앞부터 내보내기
LFU사용 횟수가 가장 적은 것 제거인기 없는 메뉴 빼기

💡 연관 사상·집합 연관 사상에서 불가피하게 사용됩니다. 아래 미니랩에서 직접 체험해 보세요!

✍️ 캐시 쓰기 정책 (Write Policy)

CPU가 캐시에 데이터를 쓸 때, 주기억 장치도 업데이트해야 할까요?

정책동작장단점
Write-Through캐시와 주기억에 동시 쓰기단순·안전, 하지만 느림
Write-Back캐시에만 쓰고, 캐시에서 나갈 때 주기억 업데이트빠르지만 복잡함 (dirty bit 필요)
캐시 = ‘미리 꺼내 둔 인기 메뉴’로 기다림을 줄입니다.

Q. CPU가 데이터를 먼저 찾는 곳?

Q. 자주 쓰는 영역만 반복 접근하는 성질?

📖 개념의 정의

보조기억은 계층 최하단 — 느리지만 대용량·저렴해요.

장치특징
HDD자기 디스크, 기계 이동, 대용량
SSD반도체, 빠르고 충격에 강함
CD/DVD광디스크, 배포·멀티미디어

HDD 접근 시간 = 탐색 시간 + 회전 지연 + 데이터 전송 시간

HDD가 데이터를 읽는 3단계
1탐색 — 헤드가 트랙으로 이동
2회전 지연 — 원하는 섹터가 헤드 아래로
3전송 — 데이터 읽기

💿 쉬운 비유

HDD = LP판 + 바늘 (원하는 곡 찾으려 팔이 움직임), SSD = 번호 누르면 바로 나오는 전자 서랍.

💾 HDD vs SSD 비교 (시험 빈출!)
항목HDD (하드 디스크)SSD (솔리드 스테이트)
저장 방식자기 디스크 (Platter)플래시 메모리 (반도체)
접근 속도느림 (탐색+회전+전송)빠름 (무기계적 접근)
내충격성약함 (회전부 있음)강함 (움직이는 부품 없음)
전력 소모많음적음
GB당 가격저렴비쌈 (점차 하락 추세)
주 용도대용량 저장 (NAS, 백업)OS 설치, 노트북, 서버

💡 SSD는 EEPROM에서 발전한 플래시 메모리 기반입니다. ROM → PROM → EPROM → EEPROM → Flash 순서로 발전했어요!

보조기억 = ‘모든 파일이 잠자는 지하 창고’입니다.

🧪 미니 실습 — HDD 접근 시간 계산

디스크 접근 시간 = 탐색 + 회전 지연 + 전송 (교재 공식 체험)

총 접근 시간 = 23.85 ms

Q. 충격에 강한 최근 저장장치?

📖 개념의 정의

  • 워드 길이 확장: 칩 여러 개를 나란히 — 한 번에 더 많은 비트
  • 워드 용량 확장: 칩 선택 신호로 주소 확장 — 저장 칸 수 증가
  • RAID: 디스크 여러 개를 묶어 속도·안전성 향상
RAID쉬운 말
RAID 0속도 UP (백업 없음)
RAID 1거울 복사 — 하나 고장나도 OK
RAID 5나눠 저장 + 복구 정보

👥 쉬운 비유

길이 확장 = 책상을 옆으로 이어 붙이기, 용량 확장 = 책장 층을 위로 쌓기, RAID 1 = 숙제를 두 명이 똑같이 복사해 둠.

확장 = ‘더 넓게·더 높게’, RAID = ‘여러 창고를 팀으로’.
